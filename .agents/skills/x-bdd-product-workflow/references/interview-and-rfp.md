# Интервью и RFP

## Минимальный шаблон драфта

```gherkin
Feature: Вход пользователя

  Как зарегистрированный пользователь
  Я хочу войти по email и паролю
  Чтобы получить доступ к своему кабинету

  Scenario: 01. Успешный вход с корректными учётными данными
    Given зарегистрирован пользователь "user@example.com"
    When пользователь вводит корректный логин и пароль
    Then пользователь получает доступ к личному кабинету
```

Структуру имени файла и сценариев проверяй по `x-bdd-godog`.

## Вопросы для бизнес-интервью

По каждой функции спроси:
- Что пользователь видит, если всё прошло хорошо?
- Что считается ошибкой ввода?
- Какие скрытые бизнес-правила влияют на исход?
- Какие пограничные случаи важны для бизнеса?
- Что должно произойти при отказе внешней системы?

## Что делать на RFP

- собрать вопросы front/back/mobile по готовому `.feature`
- продуктовые ответы внести в сценарии
- технические ответы увести в ADR, `doc.go` или другой инженерный документ

## Антипаттерны

- писать `.feature` раньше, чем понятна бизнес-цель
- смешивать несколько проверок в одном сценарии
- оставлять технические детали в шагах
- закрывать технические вопросы прямо в `.feature`
